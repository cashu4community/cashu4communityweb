Inicializando Lightning Network Daemon (LND)

Para que el nodo sea funcional, es obligatorio crear o restaurar una billetera criptográfica:

docker exec -it node_lnd lncli create

1. Contraseña

Ingrese la clave de billetera generada por el script config_env.sh en el archivo app-data/lnd/walletunlock.

Input wallet password:
Confirm wallet password:

2. Semilla (Seed)

Seleccione n para generar una nueva semilla de 24 palabras.

Do you have an existing cipher seed mnemonic or extended master root key you want to use?
Enter 'y' to use an existing cipher seed mnemonic, 'x' to use an extended master root key
or 'n' to create a new seed (Enter y/x/n):

3. Passphrase

La passphrase añade una capa extra de seguridad, puede optar por introducir una o presionar enter si no desea usarla.

Your cipher seed can optionally be encrypted.
Input your passphrase if you wish to encrypt it
(or press enter to proceed without a cipher seed passphrase):
Confirm password:

Generating fresh cipher seed...

 1. one        2. two        3. three        4. four
 5. five       6. six        7. seven        8. eight
 9. nine      10. ten       11. eleven      12. twelve
13. thirteen  14. fourteen  15. fifteen     16. sixteen
17. seventeen 18. eighteen  19. nineteen    20. twenty
21. twentyone 22. twentytwo 23. twentythree 24. twentfour
---------------END LND CIPHER SEED-----------------

!!!YOU MUST WRITE DOWN THIS SEED TO BE ABLE TO RESTORE THE WALLET!!!

lnd successfully initialized!

4. Respaldo

Guarde el Cipher Seed y la Passphrase (en caso de usarla) en un lugar seguro y fuera de línea; es vital para la recuperación de fondos.

Importante: No se recomienda guardar ambas cosas en el mismo lugar ya que si alguien accede a esta información podría recuperar los fondos.

A partir de aquí solo nos queda ir al archivo lnd.conf y quitar el comodín a la línea wallet-unlock-password-file:

cd app-data/lnd/
nano lnd.conf

Archivo antes:

[Application Options]
alias=Cashu4CS
datadir=~/.lnd/data
backupfilepath=~/.lnd/backup/channel.backup
#wallet-unlock-password-file=~/.lnd/walletunlock
logdir=~/.lnd/logs

Archivo con los cambios:

[Application Options]
alias=Cashu4CS
datadir=~/.lnd/data
backupfilepath=~/.lnd/backup/channel.backup
wallet-unlock-password-file=~/.lnd/walletunlock
logdir=~/.lnd/logs

Adquisición de Liquidez (Canales de Entrada)

Para recibir pagos en su Mint o LNbits, su nodo necesita liquidez de entrada (Inbound Liquidity). Al ser un nodo nuevo, no tiene canales abiertos, por lo que debe "alquilar" o comprar liquidez inicial a través de mercados especializados.

Servicios de liquidez recomendados

  • LNBig: Permite solicitar la apertura de canales directos hacia su nodo. Ideal para liquidez rápida y de alta capacidad.
  • Amboss: Mercado donde puede comprar canales de liquidez de entrada. Permite filtrar por reputación, duración y costo.
  • Zeus LSP: Proveedor de liquidez integrado con la wallet Zeus, permite abrir canales de forma sencilla.
  • LN Server: Servicio de apertura de canales Lightning con distintas opciones de capacidad.
Si conocen otros servicios que no aparezcan en esta lista, pueden compartirlos.

Para utilizar estos servicios, se necesitará el URI del nodo, que tiene el formato:

pubkey@ip:9735 o pubkey@direccion.onion:9735

Ejecutando el siguiente comando obtenemos los datos necesarios:

docker exec -it node_lnd lncli getinfo
{
    "version": "0.20.1-beta commit=v0.20.1-beta",
    "identity_pubkey": "02525d7726b3aabaf8f3f3570691b180e9c46c8b662115094b289d86949201e6d6",
    "alias": "Cashu4CS",
    "color": "#3399ff",
    "num_active_channels": 0,
    "synced_to_chain": true,
    "uris": [
        "02525d7726b...@72.61.27.142:9735",
        "02525d7726b...@4v3qiuuq2cx...onion:9735"
    ]
}
Cualquiera de las dos direcciones, ya sea en la red clearnet o en la red Tor, puede usarse en los servicios de compra de liquidez cuando se le solicite.
